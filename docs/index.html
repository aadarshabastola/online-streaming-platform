    Introduction

    Welcome to the Online Streaming Platform API documentation! This API allows you to interact with our streaming service, enabling functionalities such as user authentication, content retrieval, and video streaming.

    We have language examples in Shell and JavaScript. You can switch the programming language of the examples with the tabs on the right.

    All API requests should be made to the following base URL:

    http://localhost:3000

    This will obviously change based on what port you set when running the server.

    Authentication

    Register a New User

      curl -X POST "http://localhost:3000/api/users" \
    -H "Content-Type: application/json" \
    -d '{
      "username": "john_doe",
      "email": "john@example.com",
      "password": "securepassword123"
    }'

    const axios = require("axios");

axios
  .post("http://localhost:3000/api/users", {
    username: "john_doe",
    email: "john@example.com",
    password: "securepassword123",
  })
  .then((response) => {
    console.log(response.data);
  })
  .catch((error) => {
    console.error(error.response.data);
  });

    Response Example

    {
  "message": "Registration successful",
  "user": {
    "id": "c1a2b3d4-e5f6-7890-abcd-ef1234567890",
    "username": "john_doe",
    "email": "john@example.com"
  }
}

    This endpoint registers a new user in the system.

    HTTP Request

    POST /api/users

    Request Body Parameters

    Parameter Type Required Description
    username string Yes The user's chosen username.
    email string Yes The user's email address.
    password string Yes The user's password.

    Response

    • 201 Created

      • Returns a success message and the user's information.

    • Error Responses

      • 400 Bad Request
      {
    "error": {
      "code": "USER_ALREADY_EXISTS",
      "message": "A user with this email already exists."
    }
  }

    This endpoint retrieves a list of content items, with optional filtering, sorting, and pagination.

    HTTP Request

    GET /api/contents

    Query Parameters

    Parameter Type Required Description
    search string No Search term for content titles.
    genre string No Filter by genre.
    actor string No Filter by cast member.
    content_type string No movie, tv_show, or documentary.
    release_year integer No Filter by release year.
    sort_by string No title, release_date, average_rating.
    order string No asc or desc. Default is asc.
    page integer No Page number for pagination. Default is 1.
    limit integer No Number of items per page. Default is 20.

    Response

    • 200 OK

      • Returns a paginated list of content items.

    • Error Responses

      • 400 Bad Request
    {
  "error": {
    "code": "INVALID_QUERY_PARAMETERS",
    "message": "One or more query parameters are invalid."
  }
}

    Stream Content

    curl -X GET "http://localhost:3000/api/contents/{id}/stream" \
  -H "Authorization: Bearer <your_jwt_token>" \
  -H "Range: bytes=0-"

    const axios = require('axios');
const fs = require('fs');

axios.get('http://localhost:3000/api/contents/{id}/stream', {
  headers: {
    'Authorization': 'Bearer <your_jwt_token>',
    'Range': 'bytes=0-'
  },
  responseType: 'stream'
})
.then(response => {
  response.data.pipe(fs.createWriteStream('video.mp4'));
})
.catch(error => {
  console.error(error.response.data);
});

    This endpoint streams video content to the authenticated user.

    HTTP Request

    GET /api/contents/{id}/stream

    URL Parameters

    Parameter Type Required Description
    id string Yes The UUID of the content item.

    Headers

    Header Required Description
    Authorization Yes Must be Bearer <jwt_token>.
    Range No Specifies the byte range to stream. Used for partial content.

    Response

    • 206 Partial Content

      • Returns a chunk of the video content based on the Range header.

    • 200 OK

      • Returns the full video content if Range header is not specified.

    • Error Responses

      • 401 Unauthorized
    {
  "error": {
    "code": "AUTH_REQUIRED",
    "message": "Authentication is required to access this resource."
  }
}
    • 404 Not Found
    {
  "error": {
    "code": "CONTENT_NOT_FOUND",
    "message": "Content with the specified ID was not found."
  }
}
    • 416 Range Not Satisfiable
    {
  "error": {
    "code": "INVALID_RANGE",
    "message": "The requested range is not satisfiable."
  }
}

    Error Codes

    All error responses will return a standardized error format:

    {
  "error": {
    "code": "ERROR_CODE",
    "message": "Detailed error message."
  }
}

    Common Error Codes

    Error Code HTTP Status Description
    AUTH_INVALID_CREDENTIALS 401 Invalid email or password.
    AUTH_INVALID_TOKEN 401 Invalid or expired authentication token.
    AUTH_REQUIRED 401 Authentication is required to access this resource.
    USER_ALREADY_EXISTS 400 A user with this email already exists.
    CONTENT_NOT_FOUND 404 Content with the specified ID was not found.
    INVALID_QUERY_PARAMETERS 400 One or more query parameters are invalid.
    INVALID_RANGE 416 The requested byte range is not satisfiable.

    Notes and Best Practices

    • Authentication: Always ensure that you include the Authorization header when accessing protected resources.
    • Pagination: Use the page and limit query parameters to control pagination in content listings.
    Please replace {id} with the actual content ID and <your_jwt_token> with your actual JWT token.

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
